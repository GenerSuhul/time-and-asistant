# Rollback 202607230025

Deploy the previous web and DeviceGateway artifacts before changing the schema.
Stop credential commands, take a fresh database backup, and export
`credential_audit_events` plus `employee_device_credentials`; they contain the
post-migration device truth and audit history and must not be discarded.

The pre-migration employee summaries are retained in `migration_snapshots`
under `migration_key = '202607230025'`. Restore them only after reviewing any
credential changes made since the snapshot:

```sql
begin;

with snapshot as (
  select value
  from public.migration_snapshots,
       jsonb_array_elements(rows) value
  where migration_key = '202607230025'
    and table_name = 'employee_credential_summary'
)
update public.employees employee
set fingerprint_status = (snapshot.value->>'fingerprint_status')::public.enrollment_status,
    fingerprint_count = (snapshot.value->>'fingerprint_count')::integer,
    credential_status = snapshot.value->'credential_status'
from snapshot
where employee.id = (snapshot.value->>'id')::uuid;

drop trigger if exists materialize_verified_enrollment_state on public.biometric_enrollment_sessions;
drop trigger if exists initialize_employee_device_credentials on public.employee_devices;
drop function if exists public.materialize_verified_enrollment_state();
drop function if exists public.initialize_employee_device_credential_state();
drop function if exists public.record_employee_device_credential_state(uuid,uuid,text,text,uuid,uuid,text,integer,jsonb);
drop function if exists public.recompute_employee_credential_summary(uuid);
drop table if exists public.credential_audit_events;
drop table if exists public.employee_device_credentials;
alter table public.biometric_enrollment_sessions drop column if exists verified_count;

alter table public.device_commands drop constraint if exists device_commands_employee_id_fkey;
alter table public.device_commands add constraint device_commands_employee_id_fkey
  foreign key(employee_id) references public.employees(id) on delete cascade;

commit;
```

Do not run this rollback while a fingerprint capture or employee deletion is in
flight. In particular, reverting the foreign key to `ON DELETE CASCADE` restores
the old behavior in which queued physical cleanup commands can disappear with
the employee record.
