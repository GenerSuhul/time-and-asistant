# Rollback 202607220023

The pre-migration identifiers are stored in `migration_snapshots` under
`migration_key = '202607220023'` for `employee_identifiers` and
`employee_device_identifiers`.

Rollback requires first deploying the previous web, Edge Function and gateway
consumers. Restore `employee_devices.external_person_id` and employee identifier
fields from those snapshots before dropping `employees.hikvision_employee_no`.
Do not delete a numeric person from a physical device until its previous mapping
and biometric state have been verified.

Run the database rollback in one transaction after taking a fresh Supabase backup:

```sql
begin;

with snapshot as (
  select value row_data
  from public.migration_snapshots,
       jsonb_array_elements(rows) value
  where migration_key='202607220023' and table_name='employee_identifiers'
)
update public.employees employee
set employee_code=snapshot.row_data->>'employee_code',
    external_employee_id=nullif(snapshot.row_data->>'external_employee_id','')
from snapshot
where employee.id=(snapshot.row_data->>'id')::uuid;

with snapshot as (
  select value row_data
  from public.migration_snapshots,
       jsonb_array_elements(rows) value
  where migration_key='202607220023' and table_name='employee_device_identifiers'
)
update public.employee_devices link
set external_person_id=nullif(snapshot.row_data->>'external_person_id','')
from snapshot
where link.id=(snapshot.row_data->>'id')::uuid;

drop index if exists public.employees_company_hikvision_employee_no_uidx;
alter table public.employees drop constraint if exists employees_hikvision_employee_no_numeric_check;
alter table public.employees drop column if exists hikvision_employee_no;

commit;
```

Then redeploy the last pre-migration definitions of `admin_save_employee`,
`admin_commit_employee_creation_session`, `admin_stage_employee_fingerprint` and
`admin_delete_employee`. Drop `admin_enroll_employee_fingerprint`,
`allocate_hikvision_employee_no` and `hikvision_employee_no_seq` only after no
deployed consumer references them. Compare restored row counts with the two
snapshot `row_count` values before reopening employee administration.
